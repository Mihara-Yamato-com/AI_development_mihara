# レイアウト仕様

## 1. 対象画面

本仕様は、GanttForgeのメイン画面における以下のレイアウトを定義する。

- ヘッダー
- 操作バー
- ロック状態ラベル
- スケジュール表
- ガントチャート
- レスポンシブ表示

---

## 2. 基本レイアウト方針

- 画面全体は白を基調とする。
- PCの全画面表示だけでなく、PCのハーフサイズでも主要な操作が完結できること。
- ページ全体に横スクロールバーを表示しない。
- 画面幅が不足する場合は、ガントチャートの表示範囲を減らす。
- スケジュール表の幅は、画面幅に応じて伸縮させない。
- ヘッダーと操作バーは、それぞれ左右の操作グループを持つ。
- 各要素は必要以上に密集させず、業務システムとして視認しやすい余白を確保する。

---

## 3. ヘッダー

### 3.1 デザイン

- 背景色は白とする。
- ヘッダー下部に影を表示し、メインコンテンツとの境界を明確にする。
- タイトルには `GanttForge` と表示する。
- プロジェクト選択ドロップダウンには角丸を適用する。
- ヘッダー内のボタンには共通ボタンデザインを適用する。
- 各ボタンの先頭には原則としてHeroiconsを表示する。

### 3.2 左側の配置

左から以下の順番で表示する。

1. GanttForge
2. プロジェクト選択ドロップダウン
3. 新規プロジェクト
4. 削除

### 3.3 右側の配置

右寄せで以下の順番に表示する。

1. Excel
2. PDF
3. データ出力
4. データ取込

---

## 4. 操作バー

### 4.1 デザイン

- ヘッダーの下に配置する。
- 操作バーには「表示単位」「表示列」などのグループ名を表示しない。
- 各操作はボタンまたは横並びのセグメントとして表示する。
- 表示単位と表示列はドロップダウンやチェックボックスにしない。
- 画面幅が不足する場合、表示単位と表示列の一部を非表示にする。
- その他の主要操作は可能な限り維持する。

### 4.2 左側の配置

左から以下の順番で表示する。

1. ロック
2. スケジュール追加
3. 表示単位
4. 今日
5. すべて開く
6. 表示列
7. マイルストーン
8. 編集

### 4.3 右側の配置

右寄せで以下の順番に表示する。

1. タスク管理
2. マインドマップ
3. 変更履歴
4. コメント
5. メモ
6. 即時メモ
7. スナップショット

---

## 5. 表示単位と表示列のレスポンシブ制御

### 5.1 表示単位

表示候補は以下とする。

- 日
- 週
- 1か月
- 3か月

表示優先順位は以下とする。

1. 日
2. 週
3. 1か月
4. 3か月

### 5.2 表示列

表示候補は以下とする。

- 開始
- 終了
- 状態

表示優先順位は以下とする。

1. 開始
2. 終了
3. 状態

### 5.3 表示数の増減

- 画面幅が十分な場合は、表示単位と表示列をすべて表示する。
- 画面幅が不足する場合は、優先順位の低い項目から非表示にする。
- 表示単位だけ、または表示列だけを一括で消すのではなく、両グループから均等に項目数を減らす。
- 最後まで残す項目は「日」と「開始」とする。
- 非表示にした項目は、機能自体を削除せず、画面幅が戻った際に再表示する。

非表示にする基本順序は以下とする。

1. 3か月
2. 状態
3. 1か月
4. 終了
5. 週

## 6. ロック状態ラベル

- ロックボタン押下後、操作バーとスケジュール・ガントチャートの間にラベルを表示する。
- ラベルの背景色は赤系とする。
- 表示文言は以下とする。

`🔒 ロック中ー編集操作は無効です`

- ロック中は、編集・追加・削除・ドラッグ＆ドロップなどの変更操作を無効にする。
- 閲覧、スクロール、表示単位変更など、データを変更しない操作は利用可能とする。
- ロック解除後はラベルを非表示にし、編集操作を再度有効にする。

---

## 7. メインエリア

メインエリアは以下の2領域で構成する。

- 左：スケジュール表
- 右：ガントチャート

### 7.1 スケジュール表の幅

- PC全画面幅のおよそ3分の1より少し小さい幅を基準とする。
- スケジュール表の幅は固定する。
- 画面サイズを変更しても、スケジュール表の表示幅は変更しない。
- 画面幅が狭くなった場合は、ガントチャートの表示範囲を減らす。
- ページ全体の横スクロールは発生させない。
- スケジュール表とガントチャートの境界は視認できるようにする。

推奨実装例：

```css
.main-content {
  display: grid;
  grid-template-columns: var(--schedule-table-width) minmax(0, 1fr);
  overflow: hidden;
}

.schedule-area {
  width: var(--schedule-table-width);
  min-width: var(--schedule-table-width);
  max-width: var(--schedule-table-width);
}
```

### 7.2 高さの同期

- スケジュール表とガントチャートのヘッダー高さを一致させる。
- スケジュールの各タスク行と、対応するガントチャート行の高さを一致させる。
- 親タスク・子タスクの開閉によって片方だけ行位置がずれないようにする。

---

## 8. 受け入れ条件

- ヘッダーの左側と右側に指定された項目が正しい順番で表示される。
- 操作バーの左側と右側に指定された項目が正しい順番で表示される。
- 「表示単位」「表示列」という文字ラベルが表示されない。
- 画面幅が狭くなると、指定優先順位に従って表示単位と表示列が減る。
- 表示単位と表示列のどちらかだけが不自然に全削除されない。
- スケジュール表の幅は画面サイズを変更しても変化しない。
- 画面幅の変化はガントチャートの表示範囲に反映される。
- ページ全体に横スクロールバーが表示されない。
- ロック中は赤背景のラベルが表示され、編集操作が無効になる。
